# CafeOn PROJECT.md

## 1. 프로젝트 개요

**CafeOn**은 사용자의 목적, 감정, 취향, 시간대, 날씨, 방문 이력을 바탕으로 지금 가장 어울리는 카페를 추천하는 감성 카페 발견 앱입니다.

기존 카페 탐색이 지도 검색, 키워드 검색, 리뷰 확인 중심이었다면 CafeOn은 감성 사진 카드 피드와 자연어 기반 AI 추천을 통해 사용자가 조건을 직접 정리하지 않아도 상황에 맞는 공간을 발견하도록 돕습니다.

## 2. 서비스 비전

> 카페가 켜지는 순간, 나만의 무드로  
> 검색이 아닌 발견으로, 지금 나에게 맞는 공간을 잇다

단기적으로는 카페 탐색·추천 앱으로 시작하지만, 중장기적으로는 카페를 시작점으로 복합문화공간, 독립서점, 작업 공간 등 목적과 감성 기반의 공간을 발견하는 **감성 공간 발견 플랫폼**으로 확장합니다.

## 3. 문제 정의

### 사용자 문제

사용자는 카페를 찾을 때 정보가 부족해서가 아니라, 지금 나에게 맞는 공간을 고르기 어렵다는 문제를 겪습니다.

- 검색 결과와 리뷰가 많아도 내 목적과 감정에 맞는 카페를 빠르게 고르기 어렵다.
- 지도 앱은 위치 중심이라 분위기, 목적, 감성 맥락을 충분히 반영하지 못한다.
- 기존 추천은 사용자의 현재 상태, 날씨, 시간대, 방문 이력까지 연결하지 못한다.
- 카페를 발견하는 경험보다 검색하고 비교하는 피로가 크다.

### 카페 점주 문제

소형 카페는 대형 플랫폼 광고, 리뷰 수, 팔로워 수가 부족하면 노출되기 어렵습니다.

CafeOn은 카페의 감성 사진, 분위기 카테고리, 공간 목적성을 기반으로 노출 기회를 제공하여 작은 카페도 콘텐츠 기반으로 발견될 수 있게 합니다.

## 4. 핵심 타겟

### 주요 타겟

나만의 취향과 공간 감성을 중요하게 여기며, 감성 콘텐츠에 익숙한 20~30대 사용자

### 사용 맥락

- 스터디나 작업할 카페를 찾는 상황
- 데이트나 약속 전 분위기 좋은 공간을 찾는 상황
- 기분 전환, 재충전, 혼자만의 시간이 필요한 상황
- 새로운 동네에서 감성 있는 카페를 발견하고 싶은 상황

## 5. MVP 범위

### 5.1 온보딩

신규 사용자는 소셜 로그인 후 이미지 기반의 공간 분위기를 고르고 주요 동네를 설정합니다. 사용자는 태그를 직접 고르는 것이 아니라, 마음에 드는 공간 이미지를 선택하고 서비스는 이를 내부적으로 `MoodCategory`와 `UserPreferenceVector`로 매핑합니다.

온보딩 메인 문구:

> 마음에 드는 공간을 골라주세요.
> AI가 당신의 공간 취향을 학습하여 지금 가장 어울리는 카페를 추천합니다.

구성:

- 소셜 로그인: 카카오, 네이버, 구글
- 분위기 카테고리 선택: 8개 공간 분위기 카드 중 최소 3개 선택
- 카드 UI: 스톡 이미지 3분할 콜라주 + 카테고리명 + 키워드 3개
- 카드 탭: 폴라로이드 콜라주 4장, 컬러 스와치, 분위기 설명이 있는 무드보드 확대 화면 표시
- 무드보드 선택: `이 분위기 선택` 탭 후 카드 화면으로 복귀
- 동네 설정: 최대 3개 선택, 스킵 가능
- 취향 요약: 선택한 분위기 + 동네를 모두 반영한 문장형 요약 summary에 표시 
- 온보딩 목표 완료 시간: 90초 이내

분위기 카테고리 8개:

| # | 카테고리명 | 키워드 |
|---|---|---|
| 1 | 빈티지 감성 | 레트로 · 따뜻한 조명 · 아날로그 |
| 2 | 미니멀 모던 | 화이트 톤 · 여백의 미 · 자연광 |
| 3 | 감성 자연 | 초록 식물 · 자연 채광 · 테라스 |
| 4 | 아늑한 공간 | 낮은 조명 · 소파 좌석 · 포근한 패브릭 |
| 5 | 힙한 로컬 | 골목 안 작은 카페 · 독립적인 개성 · 사장님 감성 |
| 6 | 루프탑·뷰 | 탁 트인 전망 · 하늘과 맞닿은 공간 · 시티뷰 |
| 7 | 작업하기 좋은 | 넓은 책상 · 조용한 분위기 · 콘센트와 와이파이 |
| 8 | 스페셜티 커피 | 바리스타가 직접 내리는 · 원두 향 가득 · 커피 본연의 맛 |



홈 피드의 추천 이유는 온보딩 선택과 연결되어야 합니다.


### 5.2 무드 기반 비주얼 피드

홈의 기본 탭은 발견 탭입니다. 사용자는 지도나 검색창보다 감성 사진 카드 피드를 먼저 경험합니다.

- 현재 시간, 동네, 날씨를 보여주는 컨텍스트 배너
- 온보딩 선택 태그 기반 분위기 탭 필터
- 취향, 시간대, 날씨, 방문 이력, 저장 이력을 반영한 카드 피드
- 카드 탭 시 카페 상세 페이지 이동
- 카드 저장 시 컬렉션 저장

### 5.3 대화형 AI 추천

사용자가 자연어 한 줄을 입력하면 AI가 현재 컨텍스트와 취향 신호를 바탕으로 카페를 추천합니다.

예시 입력:

> 오늘 집중이 안 되는데 분위기 바꾸고 싶어

핵심 구성:

- 채팅 진입 시 날씨, 시간, 위치, 방문 이력 자동 수집
- 상단 컨텍스트 필 스트립 표시
- 자연어 입력 후 대화 흐름 안에 추천 카드 인라인 삽입
- 추천 이유 태그 제공
- 후속 조건 칩 제공: 더 조용한 곳, 창가 자리, 다른 동네 등
- 추천 카드 탭 시 상세 페이지 이동

### 5.4 카페 상세

감성 → 맥락 → 실용 정보 순서로 구성합니다.

- 히어로 이미지
- 카페명, 영업 여부, 거리
- 분위기 카테고리
- AI 추천 이유
- 운영 시간, 주소, 가격대, 시설, 혼카페 여부
- 지도 미리보기
- 저장, 공유, 길찾기 액션

### 5.5 컬렉션

사용자가 저장한 카페를 컬렉션 단위로 관리합니다.

- 기존 컬렉션 선택 후 저장
- 신규 컬렉션 생성
- 컬렉션 삭제
- 저장 취소
- 저장된 카페 롱프레스 시 방문 후 피드백 진입 가능


### 5.6 마이페이지

MY 탭은 사용자가 자신의 취향 정보, 동네 설정, 방문 이력, 알림/고객지원/정책 정보를 확인하고 관리하는 영역입니다.

MVP에서는 아래 3가지를 핵심 정보 노출 영역으로 제공하고, 각 영역은 편집 가능한 UI로 연결합니다.

1. 프로필 영역
   - 프로필 이미지, 이름, 휴대폰번호 요약 노출
   - `프로필 수정` 진입
     - 취향 설정 수정
     - 동네 설정 수정
   - `내 정보 수정` 진입
     - 이름 수정
     - 휴대폰번호 수정
     - 로그아웃
2. 내 분위기 카테고리
   - 사용자가 선택한 주요 분위기 카테고리 노출
   - 태그 추가/삭제 또는 취향 설정 화면으로 이동
3. 최근 방문한 카페
   - 최신순 2개 노출
   - 리스트 전체 보기 진입
   - 카페 선택 시 상세 페이지 이동

그 외 설정/고객지원/정책 메뉴는 리스트형 메뉴로 구성합니다.

- 알림 설정
- 1:1 문의
- 공지사항 및 이용약관
  - 공지사항: 리스트 > 상세
  - 약관 및 정책
    - 이용약관
    - 개인정보 처리방침
    - 위치정보 이용약관
    - 입점정책
  - 버전 정보

### 5.7 방문 후 피드백

방문이 추정되거나 사용자가 직접 진입한 경우에만 피드백을 요청합니다.

진입 경로:

- 길찾기 실행 후 30분~3시간 사이 앱 재진입
- 컬렉션 탭에서 저장된 카페 카드 롱프레스 후 `다녀왔어요` 선택

피드백 단계:

1. 방문 여부 확인
2. 추천 평가: 잘 맞았어요 / 그냥 그랬어요 / 별로였어요
3. 무드 확인: 실제 분위기 태그 선택, 선택 사항

## 6. 핵심 사용자 가치

- 조건을 직접 설정하지 않아도 지금 상황에 맞는 카페를 추천받을 수 있다.
- 지도 검색이 아닌 감성 카드 피드로 카페를 발견할 수 있다.
- 저장, 방문, 평가가 다음 추천에 반영되어 개인화가 강화된다.
- 작은 카페도 리뷰 수나 광고비가 아닌 공간 감성으로 노출될 수 있다.

## 10. 개인화 신호 정책

### 명시적 신호

- 잘 맞았어요: +0.4
- 그냥 그랬어요: +0.05
- 별로였어요: -0.35
- 분위기 카테고리 재선택 일치: +0.2
- 분위기 카테고리 재선택 불일치: -0.1
- 온보딩 분위기 카테고리 직접 삭제: -0.3

### 행동 신호

- 저장: +0.25
- 카드 공유: +0.2
- 길찾기 실행: +0.15
- 카드 상세 진입: +0.1
- 저장 취소: -0.2
- 카드 스크롤 패스 2초 미만: -0.05

### 컨텍스트 신호

- 날씨
- 시간대
- 요일
- 동네 반복 방문
- 방문 간격
