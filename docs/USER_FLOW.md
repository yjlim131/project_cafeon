# CafeOn — 유저 플로우

> 감성 카페 추천 앱 CafeOn의 전체 유저 플로우입니다. 온보딩 v4.2, MY 탭, 방문 피드백 플로우를 포함합니다.
> 마지막 업데이트: 2026.06 (v4)

---

## 전체 플로우 개요

```
앱 시작
  ├── 신규 → 소셜 로그인 → 취향 설정 → 동네 설정 → 홈 피드
  └── 재방문 ──────────────────────────→ 홈 피드
                                           │
                     ┌─────────────────────┤
                     │                     │
              AI 채팅 경로          피드 탐색 경로
                     │                     │
           AI 대화형 추천         무드 피드 탐색
                     │                     │
           추천 결과 카드            카드 선택
                     └──────┬──────────────┘
                             │
                       카페 상세 페이지
                             │
              ┌──────────────┼──────────────┐
              │              │              │
           공유            저장           길찾기
           (종료)            │         외부 앱 실행
                        컬렉션 저장           │
                             │       [30분~3시간 후]
                             │       앱 재진입 감지
                    ┌────────┘               │
                    │              방문 후 피드백 플로우
              롱프레스 진입  ←──────────────┘
                    │
             방문 후 피드백 플로우
                    │
          ┌─────────┴─────────┐
       Step 1             (아직 안 갔어요)
      방문 확인                │
          │               플로우 종료
       Step 2
      추천 평가
    (😊 / 😐 / 😞)
          │
       Step 3 (선택)
       무드 확인
          │
    AI 취향 벡터 업데이트
          │
     홈 피드 (개인화 루프)
```

## 네비게이션 바

| 탭 | 아이콘 | 역할 | 비고 |
|------|--------|------|------|
| 발견 | 나침반 | 무드 기반 카페 카드 피드 | 기본 탭 |
| AI | 스파클 | 자연어 대화형 추천 | **컬러 강조** |
| 컬렉션 | 북마크 | 저장한 카페 컬렉션 | 롱프레스로 피드백 진입 가능 |
| MY | 유저 | 취향 태그 · 동네 설정 · 방문 이력 · 고객센터 · 앱 설정 | |

> AI탭은 컬러로 강조합니다.
> 저장 액션은 카페 상세 페이지 하단 북마크 버튼으로 처리하며, 저장된 카페는 컬렉션 탭에서 관리합니다.

## 1단계 · 진입

### 앱 시작

| 조건 | 이동 경로 |
|------|-----------|
| 신규 사용자 | 소셜 로그인 → 취향 설정 → 동네 설정 → 홈 피드 |
| 재방문 사용자 | 홈 피드 (자동 로그인) |

---

## 2단계 · 온보딩 (신규 전용)

### Step 1 — 소셜 로그인
- 카카오 / 네이버 / 구글 계정으로 가입
- 별도 이메일·비밀번호 입력 없음

### Step 2 — 공간 취향 설정

#### 온보딩 메인 문구

```txt
마음에 드는 공간을 골라주세요.
AI가 당신의 공간 취향을 학습하여 지금 가장 어울리는 카페를 추천합니다.
```

#### 분위기 카테고리 (8개)

| # | 카테고리명 | 키워드 3개 |
|---|-----------|-----------|
| 1 | 빈티지 감성 | 레트로 · 따뜻한 조명 · 아날로그 |
| 2 | 미니멀 모던 | 화이트 톤 · 여백의 미 · 자연광 |
| 3 | 감성 자연 | 초록 식물 · 자연 채광 · 테라스 |
| 4 | 아늑한 공간 | 낮은 조명 · 소파 좌석 · 포근한 패브릭 |
| 5 | 힙한 로컬 | 골목 안 작은 카페 · 독립적인 개성 · 사장님 감성 |
| 6 | 루프탑·뷰 | 탁 트인 전망 · 하늘과 맞닿은 공간 · 시티뷰 |
| 7 | 작업하기 좋은 | 넓은 책상 · 조용한 분위기 · 콘센트와 와이파이 |
| 8 | 스페셜티 커피 | 바리스타가 직접 내리는 · 원두 향 가득 · 커피 본연의 맛 |

#### 선택 UX

- 8개 분위기 카드를 2열 그리드로 표시
- 각 카드: 스톡 이미지 3분할 콜라주 + 카테고리명 + 키워드 3개
- **카드 탭 시** → 무드보드 확대 화면 표시 (폴라로이드 콜라주 4장 + 컬러 스와치 + 분위기 설명)
- 무드보드에서 "이 분위기 선택" 탭 → 선택 완료 후 카드 화면으로 복귀
- **최소 3개 선택** 시 다음 버튼 활성화
- 선택된 태그 → AI 취향 벡터 초기값으로 저장

### Step 3 — 동네 설정
- 지도 핀 탭 또는 검색으로 동네 추가
- **최대 3개** 선택 가능
- **스킵 가능** — 미설정 시 GPS 위치 기반 카페 우선 노출.
- 관심 동네 선택 시 선택한 동네 카페를 랜덤 믹스하여 홈 피드 구성.

> 온보딩 목표 완료 시간: **90초 이내**

#### 취향 요약 화면 (동네 설정 이후)

홈 피드 진입 전 1회성 요약 화면 노출

```
회원님의 취향을 분석했어요

[선택한 카드 썸네일 3장]

빈티지 감성 · 아늑한 공간 · 작업하기 좋은

이 분위기를 좋아하시는군요.

            시작하기 →
```

> 요약 화면은 "AI가 내 선택을 이해했다"는 확인 장치입니다.
> 홈 피드 카드의 추천 이유 문구도 온보딩 선택 태그와 명시적으로 연결됩니다.
> 분위기 + 동네를 모두 반영한 요약

## 3단계 · 홈 피드 (발견 탭)

### 화면 구성
- **컨텍스트 배너**: 현재 시간 · 날씨 자동 표시. 한 줄로 간단히.
- **위치 선택 필터**: '성수동 외 N' 또는 '현재 위치'. '현재 위치로 설정' 제공
- **무드 탭 필터**: 온보딩 선택 태그 기반 자동 생성, 클릭 빈도에 따라 순서 개인화. 탭이 많을 시 스와이퍼. 스크롤 미노출.
- **카드 피드**: AI가 취향·시간·날씨·방문 이력을 종합해 순서 개인화

### 진입 경로 분기

| 경로 | 진입 방법 |
|------|-----------|
| AI 대화형 추천 | 하단 네비게이션 AI 추천 탭 |
| 무드 피드 탐색 | 카드 스크롤 · 무드 탭 선택 |

---

## 4단계 · 탐색

### 경로 A — AI 대화형 추천

1. 자연어 입력 ("오늘 집중이 안 되는데 분위기 바꾸고 싶어")
2. AI가 감정 · 날씨 · 시간 · 방문 이력 종합 분석
3. 추천 결과 카드 인라인 노출 (이유 · 무드 · 거리)
4. 후속 조건 칩 제시 ("더 조용한 곳", "창가 자리", "다른 동네")

### 경로 B — 무드 피드 탐색

1. 무드 탭 필터 선택 또는 카드 스와이프
2. 원하는 카드 탭 → 카페 상세 페이지 이동

---

## 5단계 · 카페 상세 페이지

### 표시 정보 (감성 → 맥락 → 실용 순서)

| 섹션 | 내용 |
|------|------|
| 히어로 이미지 | 감성 사진 · 스와이프로 추가 사진 |
| 기본 정보 | 카페명 · 영업 여부 · 거리 |
| 무드 태그 | 공간 분위기 태그 |
| AI 추천 이유 | "왜 지금 이 카페인지" 한 문단 |
| 운영 정보 | 운영시간 · 주소 · 가격대 · 시설 · 혼카페 여부 |
| 지도 미리보기 | 인라인 지도 · 탭 시 외부 앱 연결 |


### 하단 액션 버튼

| 버튼 | 동작 |
|------|------|
| 저장 | 컬렉션 선택 또는 새 컬렉션 생성 → 암묵적 선호 신호 반영 |
| 공유 | OS 기본 공유 시트 표시 → 링크 공유 → 암묵적 선호 신호 반영 |
| 길찾기 | 바텀시트 외부 앱 실행 · 길찾기 실행 시각 기록 |

---

## 6단계 · 행동

### 6-A. 링크 공유

1. 공유 버튼 탭 → OS 기본 공유 시트 표시
2. 공유 채널 선택 (카카오톡 · 인스타그램 · 링크 복사 · 더 보기)
3. 공유 완료 → **암묵적 선호 신호(+0.2)로 가중치 반영**
4. **피드백 요청 없음** (방문 확인 불가)

> 별도 카드 스타일 선택 화면 없이 OS 기본 공유 시트로 처리합니다.
> 공유 링크는 카페 상세 페이지 딥링크로 구성되며, 수신자가 앱 미설치 시 웹 프리뷰 페이지로 이동합니다.

### 6-B. 저장

1. 저장 버튼 탭 → 바텀시트로 컬렉션 선택 또는 새 컬렉션 생성
2. 저장 완료 → **암묵적 선호 신호(+0.25)로 가중치 반영**
3. 저장된 카페는 하단 네비게이션 컬렉션 탭에서 관리
4. **피드백 요청 없음** (방문 확인 불가)

#### 컬렉션 탭 — 롱프레스 직접 피드백 진입

저장된 카페를 다녀온 후 자발적으로 피드백을 남길 수 있는 직접 진입 경로입니다.

```
컬렉션 탭 → 저장된 카페 카드 롱프레스
    │
컨텍스트 메뉴 표시
    ├── 다녀왔어요 → 방문 후 피드백 플로우 진입 (7단계)
    ├── 카페 상세 보기
    ├── 다른 컬렉션으로 이동
    └── 저장 취소
```

> 롱프레스 피드백은 앱 재진입 감지 트리거와 무관하게 언제든 진입 가능합니다.
> 단, 취향 벡터 가중치는 동일하게 적용됩니다. 카페 무드 태그 집계에는 포함하되,
> 길찾기 이력이 없는 경우 가중치를 0.7 배율로 감산 적용합니다.

### 6-C. 길찾기

1. 길찾기 버튼 탭
2. 길찾기 실행 시각 기록 (타임스탬프 저장)
3. 외부 앱 실행 (카카오맵 · 네이버지도 · 구글맵 선택)

> 별도 알림 수락 팝업 없음. 사전 동의 없이 푸시 발송하지 않습니다.

#### 앱 재진입 감지 트리거

```
길찾기 실행 후 30분~3시간 사이에 앱 재진입
    │
방문 후 피드백 플로우 자동 진입 (7단계)
```

| 조건 | 처리 |
|------|------|
| 재진입 시간이 30분 미만 | 피드백 플로우 미진입 (이동 중으로 판단) |
| 재진입 시간이 30분~3시간 | 방문 후 피드백 플로우 진입 |
| 재진입 시간이 3시간 초과 | 피드백 플로우 미진입 · 컬렉션 롱프레스로 자발적 진입 유도 |
| 앱 재진입 없이 세션 종료 | 길찾기 실행 신호만 가중치 반영 (+0.15) |

## 7단계 · 방문 후 피드백

### 진입 경로

| 경로 | 트리거 |
|------|--------|
| 앱 재진입 감지 | 길찾기 후 30분~3시간 내 앱 재진입 시 자동 진입 |
| 컬렉션 롱프레스 | 컬렉션 탭에서 저장된 카페 카드 롱프레스 → '다녀왔어요' 선택 |

### Step 1 — 방문 여부 확인

화면 하단에서 올라오는 바텀 시트로 표시합니다.

```
[카페 썸네일]  OO 카페 · 성수동

카페온 추천으로 다녀오셨나요?

    ✓ 네, 다녀왔어요     ✗ 아직 안 갔어요
```

| 선택 | 처리 |
|------|------|
| 네, 다녀왔어요 | Step 2로 진행 |
| 아직 안 갔어요 | 플로우 종료 · 컬렉션에 저장 상태 유지 |

### Step 2 — 추천 평가 (필수)

이 추천이 나에게 맞았는지를 묻습니다. 취향 벡터 업데이트의 핵심 신호입니다.

```
카페온 추천, 어떠셨나요?

   😊              😐              😞
잘 맞았어요    그냥 그랬어요    별로였어요
```

- 3가지 반응 중 하나 탭 → 즉시 Step 3으로 전환
- 취향 벡터에 즉시 반영 · 완료 화면에서 "다음 추천에 반영했어요" 노출
- 😞 선택 시 한 가지 추가 질문: "어떤 부분이 기대와 달랐나요?" (선택)

| 반응 | 가중치 | 취향 벡터 처리 |
|------|--------|----------------|
| 😊 잘 맞았어요 | +0.4 | 해당 카페 무드 태그 가중치 상승 |
| 😐 그냥 그랬어요 | +0.05 | 변화 미미 |
| 😞 별로였어요 | -0.35 | 해당 카페 무드 태그 가중치 하강 |

### Step 3 — 무드 확인 (선택)

```
실제로 어떤 분위기였나요? (선택 · 최대 2개)

[조용한 오후]  [작업하기 좋은]  [창가 좋은 곳]
[빈티지 감성]  [브런치 감성]   [넓고 여유로운]

            건너뛰기
```

- 카페에 기존 등록된 무드 태그 목록 표시
- 실제 경험과 맞는 태그 선택 → 카페 무드 태그 집계 데이터에 반영
- 기존 태그와 다른 태그 선택 시 → 카페 데이터 보정 신호로 처리
- 건너뛰기 선택 또는 5초 무응답 시 → 완료 화면으로 이동

### 처리 결과

- Step 2 추천 평가 + Step 3 무드 태그 → AI 취향 벡터 즉시 업데이트
- 피드백 완료 후 → 취향 지도에 방문 기록 자동 추가
- "다음 추천에 반영할게요" 토스트 메시지 노출 후 홈 피드 개인화 루프로 복귀

## 8단계 · MY 탭

MY 탭은 사용자의 개인화 정보와 앱 설정을 관리하는 영역입니다. MVP에서는 아래 3개 영역을 단순 메뉴가 아닌 정보 노출형 카드 UI로 먼저 보여주고, 각 영역은 편집 가능한 화면으로 연결합니다.

### 화면 구조

```txt
MY 탭
  ├── 프로필 영역
  │   ├── 프로필 수정
  │   │   ├── 공간 취향 설정 수정
  │   │   └── 동네 설정 수정
  │   └── 내 정보 수정
  │       ├── 이름/휴대폰번호 수정
  │       └── 로그아웃
  │
  ├── 내 분위기 카테고리
  │   └── 편집 → 온보딩 분위기 선택 화면 재사용
  │
  ├── 최근 방문한 카페
  │   ├── 최신순 2개 노출
  │   └── 전체 리스트 > 카페 상세
  │
  ├── 알림 설정
  ├── 1:1 문의
  └── 공지사항 및 이용약관
      ├── 공지사항
      │   └── 리스트 > 상세
      ├── 약관 및 정책
      │   ├── 이용약관
      │   ├── 개인정보 처리방침
      │   ├── 위치정보 이용약관
      │   └── 입점정책
      └── 버전 정보
```

### 8-A. 프로필 영역

| 항목 | 내용 | 이동 |
|---|---|---|
| 프로필 이미지 | 사용자 아바타 또는 기본 이미지 | 프로필 수정 |
| 이름 | 사용자 이름 | 내 정보 수정 |
| 휴대폰번호 | 마스킹 표시 | 내 정보 수정 |
| 주요 동네 | 최대 3개 | 동네 설정 수정 |

### 8-B. 프로필 수정

- 공간 취향 설정 수정
  - 온보딩의 분위기 카테고리 선택 화면을 재사용
  - 기존 선택값은 선택된 상태로 표시
  - 최소 3개 유지
  - 수정 완료 시 `user_preferences` 업데이트
- 동네 설정 수정
  - 최대 3개 선택
  - 스킵 또는 전체 삭제 가능
  - 미설정 시 GPS 기반 우선 노출

### 8-C. 내 정보 수정

- 이름 수정
- 휴대폰번호 수정
- 로그아웃

MVP에서는 회원 탈퇴는 제외하고, 추후 정책/CS 범위에서 확장합니다.

### 8-D. 내 분위기 카테고리

- 사용자가 선택한 대표 분위기 카테고리를 칩 또는 카드로 노출
- 예: 빈티지 감성, 아늑한 공간, 작업하기 좋은
- 편집 버튼 선택 시 공간 취향 설정 수정 화면으로 이동

### 8-E. 최근 방문한 카페

- 방문 후 피드백 완료 또는 길찾기 후 방문 확인된 카페를 최신순으로 표시
- MY 홈에서는 최신순 2개만 노출
- 전체 보기 선택 시 방문 카페 리스트로 이동
- 리스트의 카페 카드 탭 시 카페 상세 페이지로 이동

### 8-F. 설정/고객지원/정책

| 메뉴 | Depth | 비고 |
|---|---|---|
| 알림 설정 | MY > 알림 설정 | 서비스/마케팅/피드백 리마인더 토글 |
| 1:1 문의 | MY > 1:1 문의 | 카테고리, 제목, 내용 입력 |
| 공지사항 | MY > 공지사항 > 상세 | 리스트/상세 구조 |
| 약관 및 정책 | MY > 약관 및 정책 > 상세 | 이용약관, 개인정보 처리방침, 위치정보 이용약관, 입점정책 |
| 버전 정보 | MY > 버전 정보 | 앱 버전, 최신 여부 |

## 추천 가중치 계산

### 레이어 1 — 명시적 신호 (Explicit)

| 신호 | 가중치 |
|------|--------|
| 방문 후 추천 평가 😊 잘 맞았어요 | +0.4 |
| 방문 후 추천 평가 😐 그냥 그랬어요 | +0.05 |
| 방문 후 추천 평가 😞 별로였어요 | -0.35 |
| 무드 태그 재선택 일치 | +0.2 |
| 무드 태그 재선택 불일치 | -0.1 |
| 온보딩 취향 태그 직접 삭제 | -0.3 |
| 😞 이유 선택 — 시끄러웠어요 | -0.15 (노이즈 태그 하강) |
| 😞 이유 선택 — 분위기 달랐어요 | -0.1 (무드 태그 보정) |

### 레이어 2 — 행동 신호 (Behavioral)

| 신호 | 가중치 |
|------|--------|
| 저장 | +0.25 |
| 카드 공유 | +0.2 |
| 길찾기 실행 | +0.15 |
| 카드 상세 진입 | +0.1 |
| 저장 취소 | -0.2 |
| 카드 스크롤 패스 (2초 미만) | -0.05 |

> 컬렉션 롱프레스 피드백(길찾기 이력 없음): 레이어 1 가중치에 0.7 배율 적용

### 레이어 3 — 컨텍스트 신호 (Contextual)

| 신호 | 역할 |
|------|------|
| 날씨 | 비 오는 날 저장 → 실내·창가 태그 가중치 상승 |
| 시간대 | 활동 패턴 많은 시간대 피드 우선순위 조정 |
| 요일 | 주말 방문 이력 → 주말 피드 가중치 상승 |
| 동네 반복 방문 | 3회 이상 → 해당 동네 카페 우선 노출 |
| 방문 간격 | 2주 이상 미방문 → 다양성 가중치 상승 |

### 핵심 원칙

1. **신호 강도는 실제 방문 여부에 비례** — 저장(+0.25) < 방문 추천 평가(+0.4)
2. **부정 신호도 동등하게 반영** — 필터 버블 방지
3. **공유·저장은 피드백 루프 제외** — 방문 미확인 상태에서 피드백 요청 불가
4. **길찾기 이력 없는 피드백은 가중치 감산** — 카페 집계 데이터 신뢰도 보정
